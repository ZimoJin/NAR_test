Golden Gate: Type IIS Primer Design with Overhang Selection and Domestication Support

Abstract: The Golden Gate module designs primers for Type IIS restriction enzyme-based assembly, enabling one-pot, ordered ligation of multiple fragments. It constructs primers with properly oriented Type IIS sites and user-defined overhangs, assists with selecting non-palindromic 4-bp junction overhangs, and supports domestication by identifying and mitigating internal Type IIS sites within fragments.

Introduction

Golden Gate assembly leverages Type IIS restriction enzymes that cut outside their recognition sequences, producing user-defined overhangs. This allows seamless, scar-minimized assembly of multiple parts in a defined order, often within a single reaction that alternates digestion and ligation.

Inputs

  • Fragments: Two or more sequences (FASTA supported) in the desired assembly order.
  • Enzyme choice: Common Type IIS enzymes (e.g., BsaI, BsmBI, BbsI) with their recognition and cut patterns.
  • Overhang preferences: Fixed user-provided overhangs or auto-selected overhang sets.
  • Primer constraints: Target core Tm and allowed primer length range for fragment amplification.

Design Strategy

Type IIS site handling

Primers are constructed to include the appropriate Type IIS recognition sites in the correct orientation, followed by the 4-bp overhang and the fragment-binding core. This ensures that, after digestion, the assembled fragments expose the intended sticky ends.

Overhang selection

For multi-fragment assemblies, the module encourages non-palindromic, low-cross-reactivity overhang sets to reduce mis-ligation. When an overhang library is available, overhangs can be selected to favor higher-fidelity ligation behavior and reduce error-prone junctions.

Domestication

Internal occurrences of the selected Type IIS recognition site within fragments can disrupt assembly. The module scans fragments for internal sites and provides guidance (and, where supported by the workflow, silent substitutions) to remove problematic sites while preserving coding sequence integrity.

QC integration

All primers are evaluated with shared thermodynamic and secondary-structure checks, enabling users to detect problematic cores (e.g., strong dimers) before ordering.

Outputs

  • Primer list: Per-fragment forward/reverse primers containing Type IIS site + overhang + core.
  • Junction map: Overhangs assigned to each junction to verify intended assembly order.
  • Domestication report: Internal site hits and recommended mitigation.
  • QC: Tm/GC and secondary-structure summaries.

Limitations and Practical Notes

  • Overhang choice can depend on context (neighbor bases, ligase, cycling conditions); empirical optimization may still be required for large assemblies.
  • Domestication recommendations assume standard coding constraints; users should confirm amino acid preservation and regulatory impacts.
  • Very short fragments may require relaxed core constraints to meet target Tm.
